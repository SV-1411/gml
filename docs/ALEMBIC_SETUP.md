# Alembic Setup - Quick Reference

## ✅ Setup Complete!

Alembic has been configured for async SQLAlchemy migrations. All configuration files are in place.

## 📋 Exact Commands for macOS Terminal

### Step 1: Install Dependencies

```bash
cd "/Volumes/Yatri Cloud/org/gml/project"
pip install -r requirements-dev.txt
```

### Step 2: Verify Alembic Installation

```bash
alembic --version
```

Expected output: `alembic 1.13.2` (or similar)

### Step 3: Check Current Database State

```bash
alembic current
```

If this is your first migration, you'll see: `(empty)` or an error (which is normal).

### Step 4: Generate Initial Migration

This will create a migration file for all 7 models (Agent, Message, Capability, Memory, Cost, AuditLog, Connection):

```bash
alembic revision --autogenerate -m "initial_migration_create_all_tables"
```

**Output:** Creates a file like `alembic/versions/XXXX_initial_migration_create_all_tables.py`

### Step 5: Review the Generated Migration

**⚠️ IMPORTANT: Always review before applying!**

```bash
# View the migration file
cat alembic/versions/*_initial_migration_create_all_tables.py

# Or open in your editor
open -a "Cursor" alembic/versions/*_initial_migration_create_all_tables.py
```

### Step 6: Apply the Migration

```bash
alembic upgrade head
```

**Expected output:**
```
INFO  [alembic.runtime.migration] Running upgrade  -> <revision_id>, initial migration create all tables
```

### Step 7: Verify Migration Applied

```bash
# Check current revision
alembic current

# View migration history
alembic history
```

## 🔄 Common Migration Commands

### Generate New Migration (After Model Changes)

```bash
alembic revision --autogenerate -m "description_of_changes"
```

### Apply Migrations

```bash
# Apply all pending migrations
alembic upgrade head

# Apply one migration at a time
alembic upgrade +1

# Apply up to specific revision
alembic upgrade <revision_id>
```

### Rollback Migrations

```bash
# Rollback one migration
alembic downgrade -1

# Rollback to specific revision
alembic downgrade <revision_id>

# Rollback all (⚠️ DANGEROUS - drops all tables)
alembic downgrade base
```

### View Status

```bash
# Current database revision
alembic current

# All migrations
alembic history

# Pending migrations
alembic heads

# See SQL without executing (dry run)
alembic upgrade head --sql
```

## ⚠️ Important Notes

1. **Always review** autogenerated migrations before applying
2. **Test on development** database first
3. **Backup production** database before migrations
4. **Never edit** existing migrations - create new ones instead
5. **Use descriptive** migration messages

## 🐛 Troubleshooting

### "Can't locate revision identified by..."

```bash
# Check current revision
alembic current

# If needed, stamp the database
alembic stamp head
```

### "Target database is not up to date"

```bash
# Apply pending migrations
alembic upgrade head
```

### "No such revision"

```bash
# View available revisions
alembic history

# Use correct revision ID
alembic upgrade <correct_revision_id>
```
